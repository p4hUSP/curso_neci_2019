Lidando com fatores

Vamos supor que estamos trabalhando com dados de survey e em uma das questes temos a escala likert, em que as respostas vo do Discordo totalmente at o Concordo totalmente contabilizando um total de 6 categorias de resposta. Ocorre que para que o R entenda este tipo de dado como algo categrico, precisamos transform-los em fatores (factors).

Para fazer essa transformao, utilizaremos uma funo chamada factor() que recebe como parmetro um vetor com o que queremos transformar em categorias.

factor(c("Discordo totalmente", "Discordo parcialmente", "Indiferente", "Concordo parcialmente", "Concordo totalmente"))

Diferente de quando executamos um dado tipo texto, os dados categricos apresentam os nveis das categorias (levels). Perceba que as nossas categorias no esto de acordo com o que a escala likert pressupe e para arrumar este problema, precisamos definir os nveis atravs de um outro parmetro da funo factor() chamado levels =.

factor(c("Discordo totalmente", "Discordo parcialmente", "Indiferente", "Concordo parcialmente", "Concordo totalmente"),
       levels = c("Discordo totalmente", "Discordo parcialmente", "Indiferente", "Concordo parcialmente", "Concordo totalmente"))

Lidando com lgicos/booleanos

Valores booleanos so resultados de operaes booleanas e podem ser definidos como verdadeiros ou falsos. Por conveno, trabalhamos com os termos em ingls. Portanto, TRUE ou FALSE.

Por operaes booleanas, podemos pensar em diferentes testes que tenham necessariamente uma resposta ou verdadeira ou falsa. Por exemplo, 3 maior do que 1? Verdadeiro! E 3 menor do que 2? Falso!

Vamos realizar essas duas operaes no R.

# 3  maior do que 1?

# 3  maior do que 1?

Existem, obviamente, outras operaes. possvel tambm testar a igualdade entre dois valores com ==.

No iremos introduzir essa ideia aqui, mas caso seja de interesse tambm podemos utilizar as noes de maior e menor para textos. Voc teria um palpite de como um texto pode ser maior do que outro?

Estrutura de dados

As estruturas de dados so maneiras de organizar e guardar dados dentro da linguagem para que possamos trabalhar de forma mais eficiente. No nosso cotidiano utilizaremos trs destas estruturas: vetores atmicos, listas e dataframes; divididos em tipos de estruturas homogneas e heterogneas. Mas o que isso significa? Basicamente diz respeito a quantidade de tipos de dados que ela pode receber, isto , estruturas homogneas s podem receber um tipo de dado, enquanto as estruturas heterogneas podem receber mais de um tipo de dado.

Um vetor atmico a estrutura mais bsica no R; um vetor de dados homogneo com apenas uma dimenso. Uma analogia interessante para entendermos um vetor atmico a seguinte:

Pacotes

Segundo a definio do livro R Packages, escrito por Hadley Wickham, um pacote uma unidade fundamental de compartilhamento de cdigos, dados, documentao e testes. Em outras palavras, os pacotes so colees de cdigos (funes) e dados desenvolvidos pela comunidade.

Estes pacotes ficam localizados no que chamamos de repositrio. O principal repositrio recebe o nome de Comprehensive R Archive Network, o tambm chamado, CRAN. Mas existem outros tambm, como o Github, no qual a forma de obter os pacotes muda. Mas como instalamos um pacote no R? A resposta simples, utilizamos uma funo chamada install.packages() em que o argumento principal o nome do pacote que queremos instalar.

Uma peculiaridade desta funo que por padro ela instala os pacotes que esto no CRAN, como por exemplo, vamos instalar um pacote chamado dplyr

install.packages("dplyr")

Aps instalarmos o pacote, devemos habilit-lo com uma funo chamada library()

library(dplyr)

Caso o pacote no esteja no CRAN, mas sim no Github, devemos instalar o pacote com o auxilio de uma funo chamada install_github() do pacote devtools.

install.packages("devtools")

library(devtools)

install_github("abjur/abjutils")

library(abjutils)

muito comum estabelecermos uma condio para tomar uma deciso, como por exemplo, Porcentagem de votos maior que 50% pode significar vitria, caso contrrio, derrota. if/else

A estrutura do if/else a seguinte:

if(<condicao>){
  <comando1>
  <comando2>
  <etc>
}else{
  <comando3>
  <comando4>
  <etc>
}

Podemos interpret-la da seguinte maneira: Se a condio for verdadeira, execute os comandos 1 e 2, caso contrrio, execute os comandos 3 e 4. Pensando em algo menos abstrato podemos dizer que caso o candidato para o pletio tenha recebido mais de 50% dos votos, ele est eleito, caso contrrio, ele perdeu a eleio.

porcentagem_de_votos <- 55
if(porcentagem_de_votos > 50){
  print("Eleito")
} else {
  print("No eleito")
}

for

Vamos criar o seguinte dataframe:

  • ano_da_eleicao: Ano em que ocorreu a eleio

  • porcentagem_de_votos: Porcentagem de votos do vencedor

  • tipo_disputa: Se foi uma disputa pela presidncia ou pelo governo do Estado.

tabela <- data.frame(ano_da_eleicao = seq(1955, 2015, 5), porcentagem_de_votos = runif(13, min = 40, max = 95),
           tipo_disputa = c(rep("Governo", 3), rep("Presidencia", 3), rep("Governo", 3), rep("Presidencia", 4)))

Nosso objetivo:

Criar uma coluna chamada categoria_voto, na qual, se o voto para a disputa presidencial for maior que 70%, chamaremos de super majoritrio; caso contrrio, chamaremos de normal. Caso a disputa seja para o governo do estados iremos considerar como super majoritrio um percentual maior que 80%.

Porm, vamos entender como funciona a estrutura de um for:

for (i in x){
  <comando1>
  <comando2>
}

Ns lemos o for da seguinte maneira: Para cada valor i presente na sequncia x, realize tais comandos. Ou seja, para cada linha da nossa tabela, some um ao percentual de votos recebidos.

for(i in 1:nrow(tabela)){
  print(tabela$porcentagem_de_votos[i] + 1)
}

Vamos entender algumas coisas aqui,

  • nrow(tabela) uma funo que retorna o nmero de linhas de uma tabela, ou seja, seu resultado um nmero.

  • Quando temos 1:nrow(tabela), estamos criando um vetor com uma sequncia de 1 at o nmero de linhas da nossa tabela, no caso, 13. Isso significa que nosso for vai percorrer as treze linhas da tabela.

  • o i assumir os valores de 1 at 13, por isso utilizamos tabela$porcentagem_de_votos[i], ou seja, estamos indexando as tabelas pelos valores que i assume.

Vamos agora voltar ao nosso objetivo principal:

Criar uma coluna chamada categoria_voto, na qual, se o voto para a disputa presidencial for maior que 70% chamaremos de super majoritrio, caso contrrio, chamaremos de normal. Caso a disputa seja para o governo do estados iremos considerar como super majoritrio um percentual maior que 80%.

tabela$categoria_voto <- NA # Criando uma coluna vazia
for(i in 1:nrow(tabela)){
  if(tabela$tipo_disputa[i] == "Presidencia"){
    if(tabela$porcentagem_de_votos[i] >= 70){
      tabela$categoria_voto[i] <- "super majoritario"
    } else {
      tabela$categoria_voto[i] <- "normal"
    }
  } else {
    if(tabela$porcentagem_de_votos[i] >= 80){
      tabela$categoria_voto[i] <- "super majoritario"
    } else {
      tabela$categoria_voto[i] <- "normal"
    }
  }
}

print(tabela)
